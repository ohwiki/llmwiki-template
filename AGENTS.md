# LLM Wiki 写作知识库

> 这是一个 LLM Wiki。模式来自 Karpathy 的 llm-wiki.md (https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)。
> 把这个文件丢给任何 AI Agent（Claude Code、Codex、Kiro、Cursor、OpenClaw 等），它就知道怎么干活。

## 核心思想

不是 RAG。不是每次查询都从零检索原始文档。

而是 LLM 增量构建和维护一个持久的 wiki——一组结构化、互相链接的 markdown 文件，位于你和原始素材之间。新素材进来时，LLM 不只是索引它，而是读取、提取关键信息、整合进现有 wiki——更新摘要页、修订概念页、标注新旧矛盾、强化或挑战已有的综合判断。知识编译一次，然后持续维护，不是每次查询都重新推导。

wiki 是一个持久的、复利的产物。

你几乎不自己写 wiki——LLM 写和维护所有内容。你负责提供素材、探索方向、问对的问题。LLM 做所有苦力活——摘要、交叉引用、归档、记账。

## 三层架构

```
my-writing/
├── raw/                   ← 第一层：原始素材（不可变，LLM 只读不改）
├── wiki/                  ← 第二层：知识层（LLM 拥有这一层，创建、更新、维护）
│   ├── index.md           ← 内容地图（按类别列出所有页面和摘要）
│   ├── log.md             ← 时间线（追加记录每次操作）
│   └── *.md               ← 摘要页、概念页、风格页、素材页、对比页
└── .agents/skills/        ← 第三层：Schema（跨 Agent 通用目录）
    ├── ma-sanli-writer/SKILL.md       ← 马三立风格写作规则
    ├── ma-sanli-perspective/SKILL.md  ← 马三立视角（心智模型+决策启发式）
    ├── lu-xun-writer/SKILL.md         ← 鲁迅风格写作规则
    ├── lu-xun-perspective/SKILL.md    ← 鲁迅视角（心智模型+决策启发式）
    ├── xu-zhimo-writer/SKILL.md       ← 徐志摩风格写作规则
    ├── xu-zhimo-perspective/SKILL.md  ← 徐志摩视角（心智模型+决策启发式）
    └── nuwa-skill/SKILL.md            ← 女娲（蒸馏任何人的思维方式）
```

**raw/** 是你策划的原始素材集合。文章、笔记、外部素材。不可变——LLM 从中读取但永远不修改。这是你的事实来源。

**wiki/** 是 LLM 生成的 markdown 文件目录。摘要、概念页、风格分析、素材库、索引。LLM 完全拥有这一层。它创建页面、在新素材到来时更新、维护交叉引用、保持一切一致。你读它；LLM 写它。

**.agents/skills/** 是 Schema。告诉 LLM wiki 的结构约定、写作风格规则、以及执行操作时遵循的工作流。这是关键配置——它让 LLM 成为一个有纪律的 wiki 维护者和风格一致的写作者，而不是一个通用聊天机器人。

## 三个操作

### Ingest（摄入）

用户把新素材放进 raw/，告诉 LLM 处理它。

流程：LLM 读取素材 → 和用户讨论关键要点 → 在 wiki 中写一页摘要 → 更新 index.md → 更新 wiki 中相关的概念页和素材页 → 在 log.md 追加一条记录。一份素材可能触及 5-15 个 wiki 页面。

建议一次摄入一份素材，保持参与——读摘要、检查更新、引导 LLM 强调什么。也可以批量摄入，但监督会少一些。

### Query（查询）

用户对 wiki 提问。LLM 先读 index.md 找到相关页面，然后深入阅读，综合出带引用的回答。

关键洞察：**好的回答可以作为新页面归档回 wiki。** 你要求的对比、分析、发现的关联——这些有价值，不应该消失在聊天记录里。这样你的探索就像摄入的素材一样在知识库中复利。

当用户要求写文章时，这也是一种 Query：
- 先读 index.md 找相关素材和历史文章
- 读取对应的 Skill 规则（写作用 ma-sanli-writer/SKILL.md，思维分析用 ma-sanli-perspective/SKILL.md）
- 基于 wiki 中的积累 + Skill 的风格规则写文章
- 写完后按 Skill 中的自检体系质检
- 将新文章的摘要 file back 回 wiki

### Lint（健康检查）

定期让 LLM 检查 wiki 的健康状况。查找：
- 页面之间的矛盾
- 被新素材取代的过时内容
- 没有入链的孤立页面
- 被提到但缺少独立页面的重要概念
- 缺失的交叉引用
- 可以通过搜索填补的信息空白

## 索引和日志

**index.md** 面向内容。wiki 中所有内容的目录——每个页面列出链接、一句话摘要、元数据。按类别组织。LLM 每次 ingest 后更新它。回答查询时，LLM 先读 index 找到相关页面，再深入。在中等规模（~100 份素材、几百个页面）下效果很好，不需要嵌入式 RAG 基础设施。

**log.md** 面向时间。追加记录发生了什么、什么时候——ingest、query、lint。每条记录用一致的前缀（如 `## [2026-04-17] ingest | 文章标题`），这样 log 可以用简单的 unix 工具解析。

## 页面命名约定

| 类型 | 前缀 | 用途 |
|------|------|------|
| 摘要页 | `summary-` | 一篇素材对应一页摘要 |
| 概念页 | `concept-` | 一个概念一页 |
| 风格页 | `style-` | 风格分析、迭代发现 |
| 素材页 | `material-` | 可复用的素材、案例、金句 |
| 对比页 | `compare-` | 多个事物的对比 |

## 页面格式

- 第一行：一句话摘要（不超过 30 字）
- 来源标注：`[来源：raw/文件名]`
- 页面之间用 `[[双链]]` 互相引用
- 写作相关页面标注适用类型（解意/落地/观机小记）
- 不确定的内容标注 `[待复核]`

## 绝对规则

1. raw/ 只读不改
2. .agents/skills/ 只读不改（改 Skill 的建议写在 wiki 里，由用户决定）
3. 每次改 wiki 必须更新 index.md 和 log.md
4. 双链不能断——新建页面后检查所有引用
5. 不要为了写而写——只有真正有价值的内容才回写 wiki
